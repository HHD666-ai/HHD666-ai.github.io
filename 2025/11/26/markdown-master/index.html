🏀 前言：迈向 MVP 的必经之路

基础的 Markdown 就像运球，只能保证你不丢球。但想要拿得分王，你得学会表格（数据分析）、任务清单（训练计划） 以及 提示块（裁判响哨）。

系好鞋带，我们开始。

1. 表格 (The Box Score) 📊

没有数据统计，怎么证明你是巨星？Markdown 画表格虽然有点麻烦，但效果拔群。

  • | 用来分隔列。
  • --- 用来分割表头和内容。
  • : 用来控制对齐方向（:-- 左对齐，--: 右对齐，:-: 居中）。

代码写法：

| 球员 | 场均得分(PPG) | 命中率(FG%) | 特点 |
| :--- | :---: | ---: | :--- |
| 哈登 | 36.1 | 44.2% | 后撤步三分 |
| 库里 | 30.1 | 48.5% | 超远三分 |
| 威少 | 27.2 | 42.0% | 僵尸跳投 |
实际效果：
球员	场均得分(PPG)	命中率(FG%)	特点
哈登	36.1	44.2%	后撤步三分
库里	30.1	48.5%	超远三分
威少	27.2	42.0%	僵尸跳投
2. 任务列表 (The Workout Plan) ✅
制定一个训练计划，或者购物清单。
- [ ] 表示没做（空格）。
- [x] 表示做完了（打叉）。
代码写法：
code
Markdown
- [x] 练习 500 个后撤步三分
- [x] 护理胡子
- [ ] 拿个总冠军
- [ ] 晚饭吃锅包肉
实际效果：

练习 500 个后撤步三分

护理胡子

拿个总冠军

3. Butterfly 主题专属特效 (Magic Spells) ✨
如果你用的是 Hexo Butterfly 主题（你正在用！），你有特权使用这些超级酷炫的提示块 (Admonitions)。这比普通的 Markdown 帅多了。
3.1 提示/信息 (Info)
就像教练在场边的叮嘱。
code
Markdown
:::info
💡 **登哥提示**：代码写累了记得喝水，就像比赛暂停时要喝佳得乐。
:::
实际效果：
:::info
💡 登哥提示：代码写累了记得喝水，就像比赛暂停时要喝佳得乐。
:::
3.2 警告 (Warning)
就像裁判给了一个技术犯规警告。
code
Markdown
:::warning
⚠️ **注意**：如果不写 `base: '/仓库名/'`，你的网站就会变成白屏！
:::
实际效果：
:::warning
⚠️ 注意：如果不写 base: '/仓库名/'，你的网站就会变成白屏！
:::
3.3 危险/错误 (Danger)
恶意犯规，直接驱逐出场！
code
Markdown
:::danger
🚫 **禁止操作**：不要在生产环境直接 `rm -rf /`，除非你想退役。
:::
实际效果：
:::danger
🚫 禁止操作：不要在生产环境直接 rm -rf /，除非你想退役。
:::
4. 行内代码 vs 代码块 (The Details)
有时候我们只想在句子里强调一个小战术，不需要画整个战术板。
行内代码：用反引号 ` 包裹。
例子：请执行 npm install 命令。
代码块：用三个反引号 ``` 包裹。
对比演示：
我的绝招是 StepBack3()，但如果你要看完整逻辑：
code
JavaScript
class Harden {
  attack() {
    this.dribble();
    this.stepBack();
    this.shoot();
  }
}

6. 脚注 (Footnotes) 🦶
有些话不想在正文里说，想放在最后悄悄说。
代码写法：
code
Markdown
哈登是历史上最强的得分手之一[^1]。

[^1]: 此结论由哈登本人认证，不接受反驳。
实际效果：
哈登是历史上最强的得分手之一1。
7. 折叠内容 (Hideout) 🙈
有些代码太长了，或者有些不想让人一眼看见的内容（比如剧透），可以折叠起来。这是 Butterfly 的标签插件功能。
代码写法：
code
Markdown
{% fold 点击查看我的隐藏战术 %}
其实我的战术就是：
1. 把球给我
2. 拉开单打
{% endfold %}

### 🔥 重点：如何让 Butterfly 特效（第三点）生效？

代码里提到的 `:::info` 这种写法，是 Butterfly 主题自带的 **Tag Plugins (标签插件)**。

1.  **保存**上面的文件。
2.  在终端运行 `hexo clean`。
3.  运行 `hexo g && hexo s`。
4.  打开浏览器预览。

**如果**你发现 `:::info` 没有变身成漂亮的色块，而是直接显示了文字，说明你的 Hexo 渲染器可能太老了。你需要安装一个插件：

在终端输入：
```bash
npm install hexo-renderer-markdown-it-plus --save
(或者确保你的 Butterfly 配置文件里开启了 admonition 支持)
大多数情况下，Butterfly 默认是支持 {% note info %} 这种写法的。如果 ::: 不行，你可以把上面的 :::info ... ::: 换成：
code
Markdown
{% note info %}
这里是提示内容
{% endnote %}
(这也是 Butterfly 最标准的写法，绝对管用！)
