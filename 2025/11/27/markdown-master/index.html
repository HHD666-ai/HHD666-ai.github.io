####Markdown 写作终极指南：从菜鸟到哈登级大师😎😶‍🌫️
📅 Created 2025-11-26 | 🏷️ Tags Markdown, 教程, 哈登 | 📂 Category 技术训练营

🏀 第一节：新秀赛季（基础语法）

刚进联盟（开始写博客），先把基本功练扎实。别想着一步登天，先学会运球。

1. 标题 (Headings) - 骨架与战术板😍

标题是文章的骨架。特别注意： 在 Hexo 或大多数 SSG（静态网站生成器）里，H1 (#) 通常已经被文章的大标题占用了（就像球队的老大位置只有一个），所以正文请从 H2 (##) 开始，否则页面结构会像更衣室一样乱。

❌ 不要用 H1（除非你想抢老大球权）

✅ H2 - 这是正文的一级标题（首发阵容）

H3 - 这是正文的二级标题（替补匪徒）

H4 - 再小一点的标题（饮水机管理员）

2. 强调 (Emphasis) - 暴扣与欧洲步

平铺直叙的文字像没有对抗的训练赛。你需要加点动作：🤯🤯🤯

  • 粗体 (Bold)：像是一记战斧劈扣，用来强调重点。
  • 斜体 (Italic)：像是欧洲步 (Euro Step)，灵动飘逸，用于引用或专有名词。
  • 删除线 (Strikethrough)：像是被盖帽了，或者这句垃圾话收回。

(1). 使用 Emoji 表情符号 (❌ ✅)

你截图里那些红色的叉叉和绿色的勾勾，其实不是 Markdown 语法，而是直接输入的 Emoji 表情。
怎么打出来？

  • Windows: 按快捷键 Win + . (句号)，选择表情。😘
  • Mac: 按快捷键 Control + Command + Space。
    通用方法: 直接去复制！(复制上面代码里的 ❌ 或 ✅ 粘贴进去就行)。

(2). 标准语法结构

大标题: 一个 # 加空格。😘😘😘

对勾标题: 两个 ## 加空格。😑😑😑

粗体: 用 ** 包裹文字。
斜体: 用 * 包裹文字。
删除线: 用 ~~ 包裹文字。

  • 这是战斧劈扣（粗体）
  • 这是欧洲步（斜体）
  • 这是被盖帽了（删除线）

3. 列表 (Lists) - 战术跑位

  • 无序列表：像是自由跑位，没有先后之分。
  • 使用 *、+ 或 - 都可以。
  • 记得加个空格。
  • 有序列表：像是这套战术必须按顺序执行：
  • 1.挡拆
  • 2.顺下
  • 3.空中接力

4. 链接与图片 (Links & Images) - 妙传与高光时刻

  • 链接：给队友传球。语法：[显示文本](链接地址)。
  • 图片：这是你的比赛高光集锦。语法只比链接多一个感叹号：![图片描述](图片链接)。
    哈登小贴士：在 Hexo 中，建议使用图床（如 GitHub, OSS）来存图片，否则文章加载速度会像 38 岁的老将一样慢。

⛹️‍♂️ 第二节：常规赛磨练（进阶操作）

新秀赛季结束，你已经适应了强度。现在要开始学习一些复杂的战术配合。

1. 引用 (Blockquotes) - 赛后采访

有时候你需要引用教练的话，或者名人的名言。使用 > 符号。

“永远不要低估一颗总冠军的心。” —— 鲁迪·汤姆贾诺维奇

也可以嵌套引用（套娃）：
主教练说：

这里的防守要像疯狗一样。🤣🤣🤣

> 这是第一层引用
>> 这是第二层引用

2. 代码块 (Code Blocks) - 核心战术手册

作为技术博主，代码块就是你的战术手册。这是最关键的部分。
使用三个反引号 ``` 包裹代码，并指定语言（如 python, js, java），这样会有语法高亮（Highlight）。

def 'james_harden'():
move = "Step Back 3-Pointer"
return "Nothing but net"
如果要表示行内代码（比如提到某个变量），用单个反引号：比如 console.log(‘MVP’)。

3. 表格 (Tables) - 技术统计板

记录数据最好的方式。虽然 Markdown 写表格有点像用键盘画画，但效果拔群。

| 球员 | 得分 | 助攻 | 篮板 |
| :--- | :--- | :--- | :--- |
| 哈登 | 35 | 1 | 8 |
| 恩比德 | 30 | 4 | 15 |

  • :--- 左对齐
  • :--: 居中
  • ---: 右对齐
球员 得分 助攻 篮板
哈登 35 1 8
恩比德 30 4 15

🔥 第三节：季后赛强度（高阶技巧）😭😭😭

到了季后赛，常规武器已经被研究透了，你需要一些“不讲理”的得分手段。

1. 任务列表 (Task Lists) - 训练计划表

追踪你的待办事项。

- [x] 已完成的任务
- [ ] 未完成的任务

  • [x]早起投篮 500 次
  • [ ]力量训练
  • [x]观看录像分析

2. 数学公式 (LaTeX) - 数据分析

现在的 NBA 讲究魔球理论，不仅要会打球，还要懂数学。Markdown 支持 LaTeX 公式（通常需要博客插件支持，如 MathJax）。
行内公式：E=mc^2;

3. 脚注 (Footnotes) - 裁判回放

对某个词进行补充说明，但不想打断阅读节奏。
哈登最擅长的招数是后撤步[^1]。

4. HTML 标签 - 犯规战术

当 Markdown 原生语法不够用时（比如你想改变字体颜色、居中对齐），你可以直接写 HTML。这就像战术犯规，虽然不常用，但关键时刻能救命。

<center>这段文字被我强行居中了，就像奥尼尔站在禁区。</center>

<center>我是居中的文字</center>

<span style="color:red">我是红色的文字</span>

🏆 第四节：总决赛舞台（工具与避坑）(❁´◡`❁)(●’◡’●)

这一节我们聊聊装备和心态。

1. 最佳编辑器（你的战靴）

工欲善其事，必先利其器。

  • Typora：所见即所得，Markdown 界的 AJ1，经典、好用、优雅。
  • VS Code：程序员标配，插件丰富，像是高科技的 LeBron 系列球鞋，功能强大。
  • Obsidian：双向链笔记，适合构建知识库，属于策应型后卫的最爱。

2. 常见“失误” (Common Mistakes)

  • 空格大法：Markdown 中，符号后面通常要加空格。比如 ## 标题 对，##标题 错。这就像运球不看人，容易失误。
  • 换行陷阱：在 Markdown 源码里换一行，显示出来通常还是同一行。想换行？打两个空格再回车，或者直接空一行（分段）。
  • 中英文标点：代码块里的符号必须是英文半角！用中文的 （） 写代码，报错会让你怀疑人生。

3. Hexo 博客专属配置 (Front-matter)

在 Hexo 文章的最顶部，那两行 — 之间的内容，叫 Front-matter，这就像是球员的球衣号码和注册信息。

title: Markdown 写作终极指南：从菜鸟到哈登级大师
date: 2025-11-26 13:14:00
tags: [Markdown, 教程]
categories: 技术杂谈
cover: https://example.com/bg.jpg

⏱️ 加时赛：总结🫨🫨🫨🤖

恭喜你！读到这里，你已经掌握了 Markdown 99% 的技巧。剩下的 1%，就是像科比一样，在凌晨四点的洛杉矶（或者你的书房）不断练习。
Markdown 的魅力在于专注于内容本身，丢掉花里胡哨的排版烦恼。现在，拿起你的键盘，开始你的表演吧！
